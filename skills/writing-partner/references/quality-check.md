# 四层分层质检体系

借鉴软件工程的测试金字塔理念，从最基础的硬性规则到最主观的活人感判断，层层递进。每一层都有明确的通过标准和修复指引。只有四层全部通过，文章才算达标。

质检在共创流程的 Step 6 和打磨流程的 Step 5 执行。按已选的内容类型（公众号长文 / 商务工作）执行对应分支。

执行 L4 或打磨返工时，先加载 `human-rhythm.md`。公众号长文使用其选题、协作边界、人味来源和节奏方法；商务工作使用人味来源、负责人真实表达和返工动作清单。L4-5 人味正向检查需要对照素材池（`99-materials/<项目>/一手素材.md`）；返工参照分支化——公众号对照 `style-examples.md` 公众号长文示例节，商务工作对照 `human-rhythm.md` 负责人真实表达与人味来源（商务示例库待补，不拿公众号示例校准商务文档）。

---

## L1 硬规则（自动扫描层）

类似代码的语法检查。任何一项不通过就必须修复，没有例外。

### L1-1 AI 套话扫描

全文搜索以下词汇，出现则必须替换：

- "综上所述" / "总的来说" → 用具体回扣句
- "值得注意的是" / "不难发现" → 删掉，直接说
- "首先...其次...最后" → 删掉，直接按逻辑顺序写；跳跃时补论证链或章节硬切，不补转场句
- "让我们来看看" / "接下来让我们" → 删掉
- "在当今...的时代" / "随着...的发展" → 从具体事实切入
- "说白了" → "坦率的讲"、"其实就是"
- "意味着什么" / "这意味着" → "那结果会怎样呢"、"所以呢"
- "本质上" → "说到底"、"其实"
- "换句话说" → "你想想看"、"也就是说"
- "不可否认" → 删掉，换成正面陈述

### L1-2 空泛表达扫描（商务工作重点检查）

- "取得显著进展" / "取得阶段性成果" → 写具体完成了什么
- "打下坚实基础" / "奠定良好基础" → 写具体建成了什么能力
- "系统性提升" / "全面加强" → 写具体哪个维度提升了多少
- "持续推进相关能力建设" → 写具体推进什么、什么时间完成
- "赋能" / "抓手" / "打造闭环" → 用具体动词描述具体动作

### L1-3 结构性套话扫描

- 是否出现"让我们来看看..."、"接下来让我们..."
- 是否使用"在当今...的时代"、"随着...的发展"开头
- 是否连续使用 bullet point 罗列观点超过 3 条而不展开
- 是否有大段加粗（超过 2 行）

### L1-4 术语具体性检查

- 所有提到的工具、系统、技术是否使用了具体名称
- 没有出现"AI工具"、"某个模型"、"相关技术"、"业内最佳实践"等空泛表述

### L1-5 style gate 扫描

已落盘正文必须运行：

```bash
python3 skills/writing-partner/scripts/style_gate.py <markdown-file>
```

未落盘片段按同一规则内部扫描。重点清理：

- A/B 纠偏句、递进升格句、先否定再拔高的句式
- AI 转场和说教姿态
- 平衡腔、抽象逻辑腔
- 空泛进展和万能结尾

处理规则：

- 脚本命中默认返工
- 确需保留 A/B 纠偏句时，必须在质检报告列出命中行和保留原因
- 空泛进展类结尾必须换成具体动作、待决策问题、数据缺口或责任边界
- "有一个很关键/每一步都不神秘"这类说教句式必须改成具体判断、具体流程、真实难点或业务动作

**通过标准**：L1-1 到 L1-4 零命中；L1-5 脚本通过，或仅保留已说明原因的例外。

**修复方式**：逐个替换，参考 `anti-ai-rules.md` 中的替换建议。

### L1-6 章节级输出门槛

本层检查不只在全文完稿后执行，也适用于共创流程的每节输出和打磨流程的每段改写。若单章命中以下任一情况，不允许把该章原样交给用户：

- 章节依赖 A/B 纠偏句、"关键在于""每一步都不神秘"来推进论述
- 连续三段都是抽象判断，没有事实、对象、动作、边界、代价或待确认项
- 结尾落在"持续推进、进一步完善、不断提升"而没有具体下一步
- 看起来像在解释一个道理，缺少真实问题处理感

**通过标准**：每章输出前已完成必要改写；用户看到的是改写后的正文，而不是待修的初稿。

## L2 风格一致性（模式匹配层）

类似代码的单元测试。按内容类型分支检查。

### 公众号长文分支

**L2-A1 开头检查**
- [ ] 是否从一个具体的、当下的事件/场景切入？（不是宏大叙事）
- [ ] 第一句话是否让读者产生"然后呢？"的冲动？
- [ ] 有没有使用任何教科书式开头？

**L2-A2 节奏检查**
- [ ] 段落以自然段为主，避免每个判断都独立成段？（单句段比例由 `style_gate.py` 强制把关，默认 ≤ 15%）
- [ ] 句子是否都在推进信息？（不为节奏造表演性短句或伪口语；长而平实优于短而表演——2026-07-09 A/B 盲评口径）
- [ ] 仅章节首句总判断、章节尾句冷锤句、全文核心锤子句独立成段？普通论述合并成自然段？
- [ ] 偏离主线的段落后面是否有"扣主线句"拉回来？

**L2-A3 协作边界与真实材料检查**
- [ ] 核心判断是否来自用户或已明确标注为 AI 初步建议，避免 AI 自行编造？
- [ ] 真实经历、案例、项目细节是否来自已给素材，缺失处是否标注待补？
- [ ] 知识输出方式是否自然（"用来解释当前问题" vs "教科书科普"）？
- [ ] 是否避免把具体账号的口癖、固定尾部或人格标签复制成默认文风？

**L2-A4 分支节奏检查**
- [ ] 公众号长文：段落以自然段为主，仅核心锤子句独立成段；章节硬编号；结尾用"顺序清单/冷判断/类比定调"三种合法收法之一，避免文化升华？
- [ ] 是否避免用"我也面临这个问题"这类朋友口吻伪造亲近感？
- [ ] 是否避免用"下面我来给大家科普"等教科书式预告？

**通过标准**：L2-A1 全部通过，L2-A2 至少 3/4 通过，L2-A3 至少 3/4 通过，L2-A4 按所选风格分支通过。

### 商务工作分支

**L2-B1 判断前置检查**
- [ ] 核心判断是否在文章前几段就能找到？
- [ ] 每个领域/课题是否先给结论再展开依据？
- [ ] 读者是否不需要读完全文就能知道核心观点？

**L2-B2 论证 vs 提纲检查**
- [ ] 每个 point 是否都有实质信息（删掉后会丢失信息）？
- [ ] 读完每个 point 是否知道作者的判断是什么？
- [ ] 有没有只是空壳标题的段落？

**L2-B3 数字/数据具体性检查**
- [ ] 关键论述是否有具体数字支撑（成本、时间、人力、百分比）？
- [ ] 是否避免了空泛描述（"显著提升" vs "提升了30%"）？

**L2-B4 亮点+不足双面检查**
- [ ] 是否同时列出了亮点和不足？
- [ ] 不足部分是否写了具体问题，避免只写"仍有提升空间"？

**通过标准**：L2-B1 全部通过，L2-B2 至少 3/4 通过，L2-B3 通过，L2-B4 通过。

## L3 内容质量（深度审查层）

类似代码的集成测试。

### L3-1 观点支撑检查
- [ ] 每个核心观点是否都有具体证据/案例/数据支撑？
- [ ] 有没有空泛的观点只有论断没有例证？

### L3-2 反驳回应检查
- [ ] 在讲核心观点时，是否考虑了主要反对意见？
- [ ] 是否有对替代方案的分析（为什么不选 B）？

### L3-3 完整性检查

**公众号分支**：
- [ ] 是否用顺序清单、冷判断或类比定调收束，而不是文化/哲学/历史升华？
- [ ] 每章是否有冷判断或行动落点，避免空泛呼吁？
- [ ] 如使用回环呼应，是否采用冷判断回扣而非情绪型意象？

**汇报分支**：
- [ ] 判断链是否完整（阶段→判断→依据→约束取舍→动作）？
- [ ] 是否写清楚了待共识点和未决事项？
- [ ] 未来目标是否有时间节点和阶段划分？

### L3-4 信息层次检查

**公众号分支**：
- [ ] 知识是"随手掏出来"还是"教科书科普"？
- [ ] 引用是否自然融入论述？

**汇报分支**：
- [ ] 是否区分了三层内容：已确认事实 / 基于事实的推导判断 / 仍待验证的假设？
- [ ] 行业参照是否写明了具体公司和做法，避免只写"参考业内实践"？

**通过标准**：L3-1 和 L3-2 必须全部通过，L3-3 和 L3-4 中至少通过对应分支的所有相关项。

## L4 活人感终审（最终人格层）

最重要也最主观的一层。以读者视角通读全文，回答核心问题。

### 核心问题

**公众号**："读完这篇文章，我感觉是一个有见识的人在认真跟我聊一件打动他的事，还是一个 AI 在给我输出信息？"

**商务工作**："读完这份材料，我感觉是一个有判断力的技术 Leader 在做决策汇报，还是一个 AI 在堆砌信息？"

### 负责人通读规则（所有写作文档强制）

通读时把作者代入为真实负责人，避免代入为"写作助手"：

- 这句话是否像负责人会在会上说出口？
- 这段有没有真实约束、代价、责任边界或下一步动作？
- 如果把"AI / Agent / 平台 / 能力"这些词去掉，段落还剩下具体信息吗？
- 是否为了显得全面，把每个判断都写成"方向成立，但仍需完善"？
- 每章是否至少有一个具体业务对象、具体动作、具体数据、具体例子或具体待决策问题？

任一章节如果连续 3 段都是抽象判断句，必须补事实、动作、边界或删减。

### 感知维度

**L4-1 温度 / 判断感**
- 公众号：情绪表达是体感记忆（"我当时就愣住了"）还是知识性描述（"我感到非常震撼"）？
- 汇报：判断是否有"这个人真的想过这件事"的感觉，还是在堆砌正确但空洞的结论？

**L4-2 独特性**
- 这篇文章/材料是否有"只有这个人才会写出的角度"？
- 还是换一个 AI 也能写出差不多的东西？

**L4-3 姿态**
- 公众号：语气是不是"有见识的普通人在分享"，有没有导师讲课感？
- 汇报：语气是不是"负责人在做判断汇报"，有没有信息堆砌感？

**L4-4 心流**
- 从头到尾读，有没有哪个地方注意力断掉了？需要回头理解逻辑？
- 如果有，那个地方就是需要修复的节奏问题。

**L4-5 人味正向检查（两分支，2026-07-09 增）**
- 在场感：素材池有料时，正文消耗了吗？细节颗粒是亲历级（时点、误判、返工次数）还是转述级？（对照素材池亲历关系标注："听说·二手"条目不得写成现场颗粒）
- 不确定：归因/未来/边界三类真实边界处有没有作者层面的不确定？能否指回素材池"没把握"条目？有没有口癖化滥用（同一形态反复出现）？
- 节奏：有没有为节奏制造的表演性短拍句、提炼鼓点句、伪口语（"人就往后退""就一句话""脑子里"——2026-07-09 A/B 实证为 AI 味）？长句只在信息推进停滞时返工；亲历章节是发现顺序还是报告倒叙？
- **溯源复查**：每处亲历级细节能否指回素材池条目编号？多轮打磨后必须重查一遍——句子合并移位会造成溯源衰减；用户打磨中口述的新经历应已回流素材池，而不是被当成"查无此项"砍掉
- 返工参照：公众号对照 `style-examples.md` 公众号长文示例节的正反例改写；商务工作对照 `human-rhythm.md` 负责人真实表达（不拿公众号示例校准商务文档）

**通过标准**：整体感觉"这像是真人写的"。如果任何一项让你觉得"AI 味太重了"或"干净但没有人"，就需要返工。

### L4-6 返工动作映射

发现 AI 味后，不要只写"需要更自然"。必须把问题归类为下面至少一类，并给出对应修复动作：

| 问题类型 | 判断信号 | 修复动作 |
|----------|----------|----------|
| 缺真实判断 | 只有资料归纳，没有作者立场 | 先补一句核心判断，再补依据 |
| 缺具体细节 | 只有概念，没有人、事、数、时间、动作 | 补真实案例、业务对象、数据或现场细节；没有就标注待补 |
| 缺约束代价 | 方案像无成本成立 | 补资源、时间、组织、风险、失败条件或适用边界 |
| 节奏平铺 | 句式等长、过渡套话多、读者一路滑过去 | 删除套话，增加短句断点或章节硬切 |
| 偏离主线 | 背景或案例讲完没回到核心判断 | 补一句扣主线句，说明它支撑哪个判断 |
| 教科书科普 | 理论先行，像百科插入 | 从具体问题切入，把知识当解释工具 |
| 假完整 | 所有问题都被写成已解决 | 标注待确认、待共识、阶段性判断 |
| 假共鸣 | 强行写"我们都..."或朋友式亲近 | 换成具体读者处境；公众号长文和商务工作直接删除 |
| 缺在场感 | 细节全是转述级，看不出做过 | 从素材池取亲历条目补时点、误判、返工次数（无人称）；无料显式降级，不硬造 |
| 句级表演腔 | 为节奏造的短拍句、鼓点句、伪口语 | 换回平实的信息句和操作句；节奏问题靠删冗余不靠造短句 |

**修复方式**：先补判断和事实，再调语气。公众号长文优先补主线、真实细节和节奏；商务工作优先补负责人判断、约束代价和下一步动作。

## 质检输出格式

完成四层质检后，输出一份简洁的质检报告：

```
## 质检报告

**L1 硬规则** ✅/❌
- AI 套话：X处命中（已修复/待修复）
- 空泛表达：X处命中（已修复/待修复）
- 结构套话：X处命中（已修复/待修复）
- 术语具体性：✅/❌
- style gate：通过 / X处命中（已保留并说明原因 / 已改写 / 待修复）
- 章节级输出门槛：✅/❌（是否还有未过闸门的章节）

**L2 风格一致性** ✅/❌
- [按对应分支列出各项检查结果]

**L3 内容质量** ✅/❌
- 观点支撑：✅/❌
- 反驳回应：✅/❌
- 完整性：✅/❌
- 信息层次：✅/❌

**L4 活人感** ✅/❌
- 温度/判断感：✅/❌（具体段落：...）
- 独特性：✅/❌
- 姿态：✅/❌
- 心流：✅/❌（断点位置：...）
- 负责人通读：✅/❌（抽象判断过密的章节：...）
- 人味正向：✅/❌（在场感消耗 / 不确定落点 / 节奏 / 溯源复查；素材池无料时注明"在场感素材缺失"）
- 返工动作：缺判断 / 缺细节 / 缺约束 / 缺在场感 / 节奏平铺 / 句级表演腔 / 偏离主线 / 教科书科普 / 假完整 / 假共鸣（列出命中的 1-3 项）

**总评**：4层全部通过 / X层需要返工
**修复优先级**：[列出最需要修复的 1-3 个具体问题]
```
