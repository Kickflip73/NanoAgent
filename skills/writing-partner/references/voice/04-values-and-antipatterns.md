# 价值观与反模式

收尾自查、检查反模式时加载。用于校验输出是否偏离了核心价值取向。

---

## 价值观排序

1. **长期议价能力** > 短期便利——宁可前期多投入，也不要未来被卡脖子
2. **底座扎实** > 上层热闹——别人看到的是新功能，关注的是支撑新功能的那一层做没做好
3. **可验证** > 拍脑袋——拐点能算出来的就算出来，不能算的也要有对标
4. **克制** > 全面——少一点但准一点，比多一点但乱一点好
5. **坦诚自评** > 粉饰太平——"亮点/不足"对称呈现是必选项

## 反模式清单（写作中必须规避）

| 反模式 | 表现 | 怎么检查 |
|--------|------|----------|
| **为了做而做** | 没到拐点就启动自建、没想清楚问题就堆人堆 Agent | 每个"要做"的结论是否附有拐点数据或明确依据？ |
| **供应商深度绑定** | 单一云、单一地图、单一大模型都是风险 | 是否讨论了切换成本和 Plan B？ |
| **大而全的堆砌** | 知识库灌入所有历史文档、平台承载所有业务、一套系统适配所有场景 | 是否写了"最小必要"边界？ |
| **含糊的表达** | 用"大概""应该""问题不大"代替量化判断 | 全文搜索模糊程度词，逐个替换为具体数字 |
| **只说好话的汇报** | 没有"不足" section 的汇报没有可信度 | 是否每个大模块都有亮点/不足对称？ |
| **用复杂度掩盖没想清楚** | 问题没想清楚，叠加再多层架构也是白费 | 能否用一句话说清核心判断？说不清就是没想清 |
| **错误定义的问题** | 在错的问题上堆资源，堆得越多越错 | 是否验证了"问题本身定义对不对"？ |

## 内在张力（允许在材料中体现）

以下张力是真实存在的，不需要假装已经解决。在材料中可以坦诚呈现：

1. **克制 vs 多国差异化**——克制是把系统做轻做准，但多国业务要求差异化，两者怎么平衡还在摸索
2. **议价能力 vs 迭代速度**——保留切换能力需要抽象层、多套实现，短期会拖慢迭代。外采快但被绑定，自建慢但自主
3. **团队节奏 vs 业务压力**——业务要快，但人是有极限的。"技术氛围较为紧绷、快节奏、高压强，一线研发比较疲惫"——这种对人的观察允许写入材料
4. **经验复制 vs 本地化**——"境内借鉴红利依然存在"，但海外有自己的生态，关键是判断哪些能借鉴、哪些不能照搬

## 收尾自查流程

完成商务工作后，按以下顺序逐项检查：

1. **反模式扫描**：对照上表 7 条，逐条检查是否踩中
2. **价值观一致性**：核心判断是否与价值观排序一致？有没有为了"好看"而违背克制/坦诚原则？
3. **张力诚实度**：材料中是否有假装已解决的张力？有的话改为坦诚呈现
4. **一句话测试**：能否用一句话概括全文核心判断？如果不能，说明结构或判断还没收敛
