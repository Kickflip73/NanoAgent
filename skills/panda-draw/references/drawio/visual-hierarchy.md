# 视觉层次工具箱(layout × palette 通用)

baoyu-infographic 跨多个 style 共享的"视觉层次"技巧,矢量图能复现的部分整理在这里。所有 layout × palette 组合共用,Step 3 生成 XML 时必读。

## 三件套

drawio 矢量图的"层次感"主要靠三件事:**描边粗细 / 字号节奏 / 强调标记**。颜色和形状是 layout × palette 已经定的,这里讲**怎么让信息有主次**。

---

## 一、描边粗细(连线和边框)

### 连线 strokeWidth 三级

```text
主流程主链路:  strokeWidth=2.5   (粗,视觉主轴)
辅助 / 普通连线: strokeWidth=1     (细,占比 80%+)
强调 / 异常 / 警告: strokeWidth=3   (最粗,整图 ≤2 处)
```

**判断什么是"主流程"**:plan 文件「布局决策」段标的"主流向"上的边,从起点 → 终点的最长链。其他都是辅助。

### vertex 描边粗细

绝大多数 vertex 用 palette 默认描边宽度即可。**例外**:

- ikea-minimal palette 全图细描边 1px
- subway-transit palette 全图粗描边 2.5px
- 核心节点(整图最重要的 1-2 个)可加 strokeWidth=2 + fontStyle=1(粗体)

---

## 二、字号节奏三级(标题 / 区块 / 正文)

```text
画图主标题(画布顶,可选):  fontSize=18  fontStyle=1
分组框 / 区块标题:          fontSize=14-15  fontStyle=1
正文卡片标签:               fontSize=12-13  常规
图例 / 副标题 / 备注:       fontSize=11  常规  fontColor=<palette 副标题字>
```

**禁止**:全图所有标签都用同一字号 — 这是"乱"的根源。

**关键节点强调**:核心节点(plan 标的"主流向起点/终点")的 fontStyle=1(粗体),其他常规。强调字体不超过 3 个节点。

---

## 三、强调标记(整图 ≤2 处)

**真正需要用户注意的地方**才用强调标记。一张图最多 2 处强调,多了就失效。

### 强调手段(挑 1 个,不混用)

- **粗描边**:strokeWidth=3 + 加粗字体
- **强调色**:用 palette 的"重点/风险"色填充(如 classic-soft 橙、morandi-pastel 主题棕)
- **警示符号**:加一个小 emoji / shape 库的警告标(`⚠️` `★` 等)在节点角落

### 何时用强调

- 异常分支 / 失败回滚 / 高风险节点
- 关键决策点 / 转折点
- 必须人工介入 / 需要审批的步骤

**不该用强调**:整张图都很重要时 — 那就降级,选最重要的 1-2 处。

---

## 四、空白节奏

**留白是层次感的一半**。规则:

- 画布四周 padding **≥40px**(不要让节点贴边)
- 同组节点间距 16-20px
- 组与组间距 40px+
- 主链路通道 ≥30px(连线两侧不要被节点贴脸)
- 单页节点占比 < 60% 画布面积(超过自动走"总览页 + 细节页")

**禁止**:为了填满画布拉大节点 / 缩小间距 — 紧凑感和"挤"是两回事,前者来自尺寸统一 + 对齐,不是密度。

---

## 五、对齐(隐性层次)

- 同行节点 y 坐标完全相同(±0,不要 ±2)
- 同列节点 x 坐标完全相同
- 分组框内节点从 `(16, 32)` 起排,严格网格
- 标题、图例、节点的左边界对齐(`align=left` 时)或居中对齐(`align=center` 时)

LLM 生成 XML 时容易出现 `y=156` `y=158` `y=160` 这种"差几个像素"的错位 — bbox 草排时就严格取整为 10 的倍数,生成时不偏移。

---

## 六、图例(强制规则)

**有彩色分类必带图例**。图例位置:

- 横版:右上角或底部居中
- 竖版:底部居中或右侧

图例项 token(common-rules.md 已定):

```text
rounded=0;whiteSpace=wrap;html=1;fontSize=11;fontColor=<palette 正文字>;  (fill/stroke 用对应 palette 语义色)
```

每个图例项 = 一个小色块(16×16)+ 一行文字(说明这个色代表什么)。图例项之间间距 6px。

---

## 何时调 Codex 商量

当你的 plan 里同时出现:
- 节点数 ≥15
- 颜色 ≥4 种
- 连线 ≥20 条
- 跨组/异常/回路边 ≥3 条

→ 视觉层次容易乱,**Step 1 自动选 layout × palette 时建议走 Codex 商量**(把内容描述 + 节点统计发给 Codex,让它建议是否换布局、要不要拆页)。
