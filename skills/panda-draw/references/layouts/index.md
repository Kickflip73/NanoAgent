# Panda Draw Layout Index

Use these bundled layouts with the fixed `morandi-journal` style.

| Content shape | Layout file | Default aspect | Notes |
|---|---|---|---|
| Multiple modules, systems, or topic overview | `bento-grid.md` | 16:9 | Best general default for architecture summaries and multi-part explanations. |
| High-density guide, checklist, many points | `dense-modules.md` | 3:4 or 4:5 | Best for knowledge cards, complete guides, and social-post style dense visuals. |
| Central concept with related parts | `hub-spoke.md` | 16:9 or 1:1 | Use for maps of concepts, teams, ecosystems, or capabilities around one core. |
| Process, workflow, timeline | `linear-progression.md` | 16:9 | Use for sequences, milestones, pipelines, and step-by-step explanations. |
| A vs B, before/after, tradeoffs | `comparison-matrix.md` | 16:9 | Use for multi-factor comparison and decision tables. |
| Layers, hierarchy, architecture levels | `hierarchical-layers.md` | 16:9 | Use for layered systems, priorities, organizational levels, or influence rings. |
| Data access, pipelines, component anatomy | `structural-breakdown.md` | 16:9 | Use for system components, data gateways, exploded views, or internal structure. |

Selection rule: choose the layout that matches the source information structure, then apply `morandi-journal` as the visual style. Do not introduce unbundled layout names unless the user explicitly asks for an experimental layout.
