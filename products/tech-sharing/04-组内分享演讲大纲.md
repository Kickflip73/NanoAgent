# 从 MCP 到可恢复智能体：组内技术分享演讲大纲

建议时长：18 分钟；预留讨论：2 分钟

## 分享目标

用一个统一框架解释 2025—2026 年 Agent 工程的三条主线：MCP 解决“怎么连接”，运行时解决“怎么可靠执行”，Agentic RAG 解决“怎么获得合适证据”。听众结束后应能判断新能力应该落在哪一层，以及一次小范围验证如何启动。

## 0:00—1:30　开场：框架很多，稳定问题只有三个

- 展示五个调研方向及 15 份资料的范围。
- 给出核心判断：模型负责推理，MCP 负责连接，运行时负责状态与副作用，RAG 负责证据。
- 提问：如果一次工具调用已经成功，但进程在记录结果前崩溃，重启后该不该再执行？

## 1:30—7:30　重点一：MCP 是连接协议，不是安全边界

### 1:30—3:30　协议结构

- Host、Client、Server 三类角色。
- Server 的 Resources、Prompts、Tools；Client 的 Sampling、Roots、Elicitation。
- stdio 与远程传输只改变连接方式，不改变授权责任。

### 3:30—5:00　为什么现在值得投入

- 2025 年底进入 Linux Foundation 旗下 AAIF。
- 官方披露 10,000+ 活跃公共 Server、SDK 月下载量 9,700 万以上。
- 主要智能体产品与 IDE 已采用，连接能力开始具备跨客户端复用价值。

### 5:00—7:30　生产风险与我们的边界

- NSA 2026 报告中的四类高相关风险：隐式信任、工具/参数注入、粗粒度权限、审计与 Token 生命周期不足。
- 对照 MimiAgent：工作区 MCP 显式信任、Plan 只读 Resources、外部事件禁用 MCP、远程凭证走环境变量、MCP 调用进入执行账本。
- 现场判断题：邮件发送能力应该直接做内置 Tool、MCP Server，还是隔离 Connector？答案取决于凭证边界、事务确认和部署位置。

## 7:30—12:30　重点二：智能体运行时的核心是恢复语义

### 7:30—9:00　三类框架放到同一张图里

- OpenAI Agents SDK：轻量模型循环、Handoff、Guardrail、Tracing。
- LangGraph：显式状态图、Checkpoint、HITL、durable execution。
- CrewAI：Crews 表达协作，Flows 表达确定流程。
- 选型口径：流程是否长时、状态是否显式、失败后从哪里恢复、是否需要人工审批。

### 9:00—11:00　一次可靠 Run 的状态机

- `accepted → running → tool side effect → completed/failed/interrupted`。
- Session owner 和 runId 防止旧 Run 覆盖新状态。
- ExecutionLedger 处理“成功结果可重放，未知结果不自动重试”。
- Goal/Plan/Checkpoint 分工：生命周期、步骤视图、恢复点。

### 11:00—12:30　Multi-Agent 的边界

- 主 Agent 持有 Session 和最终答案，Worker 只做有界任务。
- 并行前检查依赖与文件路径冲突。
- 多 Agent 的收益来自上下文隔离和并行，不来自角色数量。

## 12:30—16:00　重点三：RAG 应按查询难度升级

### 12:30—14:00　四层检索路径

- L0：词法 + 向量混合检索。
- L1：HyDE、改写、rerank 等查询增强。
- L2：GraphRAG 处理关系密集、多跳和全局归纳。
- L3：Agentic RAG 负责查询分解、跨源选择、证据反思与停止判断。

### 14:00—16:00　投入拐点

- GraphRAG 的代价：实体关系抽取、图更新、社区摘要与额外查询成本。
- Agentic RAG 的代价：更多模型调用、更高尾延迟、更难复现。
- 推荐升级条件：基础检索指标已稳定，失败样本明确集中在多跳或跨源问题，新增层能在固定评测集上带来可测收益。

## 16:00—18:00　行动建议与讨论

### 建议的两周验证

- 第 1 周：建立 30—50 条真实任务评测集，覆盖普通检索、关系查询、外部工具和副作用任务。
- 第 1 周：为现有 MCP Tool 补充能力清单，标注读写属性、权限、幂等键、超时和未知结果处理。
- 第 2 周：选择 10 条多跳失败样本，对比混合检索、GraphRAG/关系索引、Agentic 路由三种实现的质量、时延和调用成本。
- 第 2 周：用 3 个故障注入场景验证 Run 恢复：工具成功后崩溃、超时结果未知、旧 Run 晚到写入。

### 讨论问题

1. 我们当前最需要标准化的是 MCP Server 接入规范，还是 Host 侧统一权限与审计？
2. 哪类任务已经达到自主检索循环的投入拐点，证据是什么？
3. Multi-Agent 的并行收益由什么指标证明，评审吞吐是否会成为新瓶颈？
