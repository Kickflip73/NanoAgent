# Mimi Mail 历史邮箱闭环实施计划

日期：2026-07-15
状态：已完成
关联调研：`20260715-Mimi-Mail历史邮箱闭环-调研.md`

## 目标

让 Mimi 能在 owner 明确选择的 Mail 账号和邮箱路径中检索历史邮件，并继续完成读取、附件、回复和整理动作。

## UI 变动检测

涉及 UI 变动：否
预览状态：不适用

## 实施步骤

1. 把邮箱路径校验抽成通用函数；`search_messages` 接受可选 `mailboxPath`，并增加复用同一路径的 `search_mailbox_messages` 目录别名，让旧安装通过 action catalog 增量同步发现能力；有路径时禁止 target=`*`。
2. Node action validator 增加可选 `source:{account,path}`，只向需要按 message ID 定位的动作传递。
3. JXA `searchMessages` 在无路径时保持统一收件箱，有路径时用现有 `findMailbox` 精确定位单一邮箱。
4. JXA `findMessage` 接受 source locator；read/list/save/reply/mark/flag/move/delete 全部复用。
5. synthetic Mail/JXA 与 stdio 测试覆盖归档搜索、历史邮件读取和整理、wildcard/路径/歧义边界及旧收件箱兼容。
6. 更新内置 action catalog、Connector 文档、README、总计划并运行完整 CI。
