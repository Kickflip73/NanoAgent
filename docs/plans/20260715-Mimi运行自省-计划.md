# Mimi 运行自省实施计划

日期：2026-07-15
状态：已完成
关联调研：`20260715-Mimi运行自省-调研.md`

## 任务目标

让 Mimi 能在主动例程和 owner 对话中读取自己的运行健康、积压、失败和近期状态变化，并通过同一视图供 CLI 运维检查。

## 方案概述

复用 Schema v6 现有表，由 Store 生成一个有界、去正文的只读快照。Agent Host Tool、RPC 和 CLI 都只做薄适配，不建立可观测性子系统。

## UI 变动检测

涉及 UI 变动：否
变动类型：无
涉及文件：无
预览状态：不适用

## 详细步骤

### 1. 统一快照

- 在 daemon types 定义 Activity snapshot、recent record 和 transition。
- Store 使用既有索引分别读取有限条 Event、Run、Outbox、Audit，裁剪错误并排除 payload、target、answer。
- 从统一 counts 确定 `needsAttention` 和 `workPending`，不引入主观健康评分。

### 2. Agent 自省工具

- 新增只读 `inspect_mimi_activity`，limit 限制为 1–20。
- 注册为 `state-read`、所有模式可用、非副作用工具。
- Dispatcher 与 Schedule Tools 一样按 Event Run 注入。

### 3. 控制面

- RPC 增加 `activity.get`。
- CLI 增加 `daemon activity [数量]`。
- README、架构和 Attention 文档说明主动例程可读取此视图，但工具不会暴露其他事务正文。

### 4. 验证

- Store 测试覆盖计数、needsAttention、workPending、排序、裁剪和内容隔离。
- Tool/Policy 测试覆盖参数边界、只读分类和可用性。
- CLI 测试覆盖 RPC 映射；Dispatcher 测试确认 Host Tool 注入。
- 运行完整 CI。
