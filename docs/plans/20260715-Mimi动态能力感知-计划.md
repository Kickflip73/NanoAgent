# Mimi 动态能力感知实施计划

日期：2026-07-15
状态：已完成
关联调研：`20260715-Mimi动态能力感知-调研.md`

## 目标

让 Mimi 在调用外部事务能力前了解 Connector 实时在线状态和 action 目录，并在渠道变化时动态刷新。

## UI 变动检测

涉及 UI 变动：否
预览状态：不适用

## 实施步骤

1. 在 Connector action tool 模块增加共享 bounded capability snapshot，最多 50 个 Connector、全局 100 个 action、单描述 300 字符，并返回真实 totals/truncated。
2. 增加 `inspect_mimi_capabilities` 只读 Host Tool，执行时动态查询 ConnectorManager。
3. `connector_action` 描述使用同一快照，逐项标注 online/offline/disabled；Manager 继续做最终原子状态检查。
4. Dispatcher 同时注入 capability 与 action tools；策略注册 capability tool 为所有模式可用的 `state-read` 非副作用能力。
5. 测试上下线变化、输出上限、描述状态、策略分类、Dispatcher 注入和 action 原有事务语义。
6. 更新 Connector/架构/README/总计划并运行完整 CI。
