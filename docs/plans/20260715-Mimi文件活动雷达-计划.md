# Mimi 文件活动雷达实施计划

日期：2026-07-15  
状态：已完成  
关联调研：[[20260715-Mimi文件活动雷达-调研.md]]

## 任务目标

让长期运行的 Mimi 主动感知配置目录中的新建或修改文件，并使用现有 Runtime 能力完成阅读、整理、归档和提醒。

## 方案概述

实现独立 NDJSON 文件雷达 Connector。它使用 Node 内置 `fs` 做周期性有界元数据扫描，输出稳定 Event，不读取正文、不持久化 cursor、不实现文件操作。目录配置放在独立 JSON，保持多 watch 和不同 Attention 语义清晰可见。

## UI 变动检测

涉及 UI 变动：否  
变动类型：无  
预览状态：不适用

## 详细步骤

1. 实现配置校验、`~` 路径展开、扩展名过滤和有界递归扫描。
2. 生成稳定 file activity Event，单目录错误隔离，重复轮询交给 Store 去重。
3. 实现 `scan_now`、`recent_files`、`watches` actions。
4. 增加文件雷达示例配置和 Connector 能力目录。
5. 更新 README、Connector、Architecture、Security 和 Changelog 文档。
6. 使用临时目录 fixture 覆盖过滤、深度、隐藏文件、符号链接、稳定 ID、修改事件和 actions。
7. 更新 npm package smoke 并运行完整 CI。

## 权衡与考量

- 使用轮询而不是 `fs.watch`，避免不同平台递归语义、丢事件和重连状态复杂度。
- Event 只传元数据，防止大文件进入 NDJSON 或模型上下文。
- 不增加通用 glob DSL，只支持大小写不敏感的扩展名 allowlist。
- 不在 Connector 中移动或删除文件；现有 Runtime 工具已经承担这些动作及副作用台账。
