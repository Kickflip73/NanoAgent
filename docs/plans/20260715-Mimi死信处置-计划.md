# Mimi 死信处置实施计划

## 目标

让 owner 能够通过轻量 CLI 明确修复或关闭长期运行中的失败任务，同时保持原 ID、执行台账和现有单进程架构。

## 实施

- [x] EventStatus 与 OutboxStatus 增加 `archived`，纳入终态、统计和安全历史清理。
- [x] Store 增加 Event/Outbox dead letter 的事务性 retry/archive，严格校验当前状态并记录 audit。
- [x] Digest briefing 对 archived 失败事件释放未消费摘要引用。
- [x] RPC 暴露四个窄方法；CLI 增加 `outbox`、`retry event|outbox`、`archive event|outbox`。
- [x] Outbox retry 输出结果明确声明 at-least-once 重投可能造成重复投递，不增加二次确认。
- [x] 补齐 Store、CLI、状态统计、历史保留和错误边界测试。
- [x] 更新架构、Connector 运维说明、总计划与阶段记录，运行完整 CI。

## 验收

- dead Event 使用同一 ID 回到 queued，attempts 重置，下一次 claim 从 1 开始。
- dead Outbox 使用同一 ID 回到 pending，attempts 重置，下一次 claim 从 1 开始。
- archive 后不会再被 worker claim，并可在保留期后被清理。
- 非 dead letter 操作失败且不改变数据。
- Daemon 不会自行重放任何 dead letter。
