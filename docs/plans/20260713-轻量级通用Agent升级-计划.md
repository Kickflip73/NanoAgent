# 轻量级通用 Agent 升级实施计划

日期：2026-07-13
状态：已完成
关联调研：[[20260713-轻量级通用Agent升级-调研.md]]

## 任务目标

将 MimiAgent 打磨为轻量级通用 Agent：核心能力真实可用、结构清晰、可扩展、可测试，并保持本地优先和较小依赖面。

## 方案概述

保持现有 CLI 与 OpenAI Agents SDK 主循环，按 `runtime / core / extensions / tools` 划分职责。优先补齐现有能力的协议兼容和可靠性，再增加 Goal、Resume 和单层 SubAgent，最后完善行为评测与文档。

## UI 变动检测

涉及 UI 变动：否
变动类型：CLI 逻辑与输出信息，无网页或桌面 UI
涉及文件：无前端页面文件
预览状态：无需预览

## 详细步骤

### 1. 规范运行时结构

将模型构建、基础提示和依赖装配从 Agent 主类中拆出，保留 `src/agent.ts` 兼容导出。

### 2. Skill Lite 1.0

使用 YAML Parser 和 Schema 校验，支持 location/root、按需读取资源、热重载和端到端测试。

### 3. MCP Lite 1.0

兼容 `servers`/`mcpServers`，支持 stdio/Streamable HTTP、连接容错、状态与重载。

### 4. 上下文和长程任务

按 Token Budget 管理上下文，生成结构化旧历史摘要；扩展 Plan 为持久 Goal、checkpoint、next action 和 resume。

### 5. 单层 SubAgent 与 Hooks

通过 Agents SDK `Agent.asTool()` 提供有限的 researcher/reviewer，主 Agent 保持最终控制；增加轻量生命周期事件总线。

### 6. 验证与开源交付

补齐单元测试、集成测试和 Agent 行为 Eval，更新 README、架构、配置示例、CHANGELOG，完成检查、提交和推送。

## 权衡与考量

- 不引入 LangGraph、数据库、队列和多 Agent 调度器。
- 复用 Agents SDK 的 MCPClient 与 Agent-as-tool，不重复实现协议层。
- 上下文压缩保持 provider-independent，兼容 DeepSeek。
- Goal 复用现有 Plan 存储，避免 Todo/Plan/Goal 三套重复状态。
