日期：2026-07-15

## 任务目标

以一个轻量子进程将多个 RSS/Atom 和 Open-Meteo 位置接入 Mimi，既能持续产生事件，也能由 Agent 主动刷新和查询。

## 方案概述

新增零依赖 `radar-connector.mjs`，读取一份严格、小型的 sources 配置。RSS/Atom 只解析条目身份、标题、链接、时间和摘要；天气只请求阈值判断需要的四个小时级字段。所有去重仍由 Mimi Store 完成。

## 详细步骤

1. 实现配置校验、有界 fetch 和定时轮询。
2. 实现 RSS 2.x / Atom 条目子集解析与稳定事件 ID。
3. 实现 Open-Meteo 小时预报归一化与可配置阈值检测。
4. 实现 `refresh/weather_snapshot/sources` action，增加示例配置和文档。
5. 使用本机 HTTP fixture 验证 RSS、Atom、天气、去重身份和 action。
6. 运行完整 CI 与 npm 打包验证。

## 权衡与考量

- 不增加 XML 依赖；解析范围故意限制在 feed entry 常用字段。
- 不在 Connector 内持久化 seen set；Store 已经是权威去重层。
- 不把天气阈值命中称为官方预警，保留数据来源和触发原因。
