# Mimi 运行卡死回收与优雅停机实施计划

日期：2026-07-15
状态：已完成
关联调研：[[20260715-Mimi运行卡死回收与优雅停机-调研.md]]

## 任务目标

让长期在线的单 Dispatcher 能从无输出的模型卡死中恢复，同时保证 Tool 外部事务不被 watchdog 中断、正常 Daemon 停机不消耗 Event 重试次数。

## 方案概述

在现有 Attention 配置中加入 `execution.runIdleTimeoutMs`，默认 20 分钟并支持热重载。Dispatcher 为每个 Agent Run 维护一个本地 timer；模型/Runtime 有进展时刷新，Tool 执行期间暂停。超时 abort 当前 Run 并走既有失败重试；Dispatcher 总 signal 中断则走既有无惩罚重排队。

## 详细步骤

### 1. 可热重载运行时限

**涉及文件：** `src/daemon/attention.ts`

在严格 schema 和默认配置中加入：

```ts
execution: {
  runIdleTimeoutMs: 1_200_000
}
```

范围为 1 分钟至 24 小时；status 只显示数值。旧配置缺字段时由 schema 默认值兼容。

### 2. Dispatcher 无进展 watchdog

**涉及文件：** `src/daemon/dispatcher.ts`

每次 Run 开始时读取最新配置；普通流事件或 Runtime Event 刷新 timer，`tool_called` 暂停，最后一个 `tool_output` 后恢复。超时时 abort 本次 Run，交给 `failEvent()` 退避、重试和最终升级。

### 3. 优雅停机无惩罚恢复

**涉及文件：** `src/daemon/dispatcher.ts`

catch 中优先区分紧急抢占和 Dispatcher 总 signal。后者调用 `preemptEvent()` 撤销本次 claim attempt、标记 Host Run interrupted 并立即回到 queued，不进入失败退避。

### 4. 测试与文档

**涉及文件：** `tests/daemon-attention.test.ts`、`tests/daemon-dispatcher.test.ts`、`README.md`、`docs/ARCHITECTURE.md`、`docs/ATTENTION.md`、`CHANGELOG.md`

验证默认值、热重载、模型卡死终止、Tool 期间暂停、正常停机等待 Tool 且 attempts 不增加，并运行完整 CI。

## 权衡与考量

- watchdog 使用现有 AbortSignal，不通过 `Promise.race` 遗弃仍在运行的 Agent Promise。
- Tool 执行期间暂停是副作用安全边界；Tool 自己继续使用各自的 timeout/cancel 机制。
- 20 分钟只限制“没有任何进展”，不会限制正常长任务总时长。
- 不新增 watchdog 服务、第二 worker、数据库字段或权限审批。
