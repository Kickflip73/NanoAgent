# Mimi 自主长期记忆实施计划

日期：2026-07-15
状态：已完成
关联调研：[[20260715-Mimi自主长期记忆-调研.md]]

## 任务目标

让 Mimi 无需逐次询问即可主动保存未来仍有价值的 owner 偏好、事实、决策和承诺，同时保持轻量、可追溯、幂等和向后兼容。

## 方案概述

只扩展现有 Memory schema 和 `remember` 工具：新记录使用 `recordedAt`，附带来源 Session、Daemon Event 与 trust；`confirmedAt` 作为旧数据兼容标记继续可读。MemoryStore 保持原子 JSON 和词法检索，不增加新服务或自动抽取管线。

## 详细步骤

### 1. 自主写入与兼容 schema

**涉及文件：** `src/core/memory.ts`

移除 `confirmed:true` 门槛，增加 recorded/provenance 字段和正文、总量边界；新旧可用记录统一检索，旧无标记数据仍不跨 Session 暴露。

### 2. 运行来源与 Agent 行为

**涉及文件：** `src/runtime/mimi-agent.ts`、`src/runtime/instructions.ts`、`src/core/user-intent.ts`

把当前 run 的 Event provenance 传入 Memory Tool；指导 Agent 主动保存稳定信息、不保存瞬时外部陈述或秘密、不向 owner 请求逐次确认；明确“不记住”仍由 Store Tool 确定性拒绝。

### 3. CLI、测试和文档

**涉及文件：** `src/commands.ts`、`tests/core.test.ts`、`README.md`、`docs/ARCHITECTURE.md`、`SECURITY.md`、`CHANGELOG.md`

更新 `/memories` 来源展示，覆盖自主写入、否决、旧数据隔离、来源审计、去重和边界，再执行完整 CI。

## 权衡与考量

- 不做后台自动抽取，避免每轮额外模型调用和不可控画像膨胀。
- 不自动开放旧无标记记忆，避免改变历史隐私语义。
- 保留 ExecutionLedger 与原子存储，它们属于可靠性而非审批。
