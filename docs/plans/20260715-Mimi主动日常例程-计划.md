# Mimi 主动日常例程实施计划

日期：2026-07-15
状态：已完成
关联调研：[[20260715-Mimi主动日常例程-调研.md]]

## 任务目标

让 Mimi 在没有新外部事件时也能按本地时区主动执行晨间规划、晚间收尾和 owner 自定义的日常检查。

## 方案概述

扩展现有 `assistant.json`，增加有界 `routines` 数组；AttentionEngine 在现有 briefing 检查旁生成普通 owner schedule Event，使用 Event 唯一键实现每日幂等和重启补发。继续使用同一个 Dispatcher 和 Agent，不新增数据库表、cron DSL 或工作流。

## 详细步骤

### 1. 有界配置模型

**涉及文件：** `src/daemon/attention.ts`

Routine 支持 `id`、`enabled`、`time`、可选 ISO weekday 1～7、prompt、priority、sessionKey 和 replyChannel。最多 50 条，ID 唯一，prompt 合计最多 50000 字符；旧配置自动获得默认晨间/晚间例程。

### 2. 本地时区到期与幂等 Event

**涉及文件：** `src/daemon/attention.ts`、`src/daemon/dispatcher.ts`

每轮 poll 检查星期和本地分钟，已到时点生成 `attention:routine`、`trust=owner`、`kind=schedule` Event。externalId 包含 routine ID、本地日期和时间，复用 Store 唯一键；同日重复 poll 或重启不会重复。

### 3. 测试与状态

**涉及文件：** `tests/daemon-attention.test.ts`

覆盖默认/旧配置、工作日、未到期、禁用、同日幂等、跨日、热重载、稳定 Session/reply route、prompt 上限和 status 不泄露正文。

### 4. 产品文档

**涉及文件：** `README.md`、`docs/ATTENTION.md`、`docs/ARCHITECTURE.md`、`SECURITY.md`、`CHANGELOG.md`

记录默认例程、配置结构、补发语义、与 briefing/Schedule/Standing Orders 的分工。

## 权衡与考量

- 只支持每天本地时间和 weekday，不实现任意 cron。
- 默认例程提供开箱即用的主动性，同时可通过 `enabled:false` 或删除关闭。
- Routine prompt 是本机 owner 配置，不包含外部 payload；外部信息仍由 Agent 通过 Connector 工具按需读取。
