# 任务持续恢复与上下文管理实施计划

日期：2026-07-14  
状态：已完成  
关联调研：`docs/research/20260714-任务持续恢复与上下文管理-调研.md`

## 任务目标

让 MimiAgent 在普通任务、Goal 与 Ultra Team 三种场景下，都能保存最近执行进展；切换会话或重启后识别中断点并通过 `/resume` 继续；同时建立不破坏原始 transcript 的分层上下文压缩。

## 方案概述

保持现有 CLI 多轮调度与 SDK 单轮 Runner，不引入通用状态机框架。扩展 `FileSession` 元数据，使 transcript、运行检查点和上下文档案原子保存。`MimiAgent` 负责单轮生命周期，CLI 只消费事件和展示。`ContextManager` 负责生成有效历史视图和压缩计划。

## 详细步骤

### 1. 扩展 Session 元数据

**涉及文件：** `src/core/session.ts`

- 新增 `RunCheckpoint`：运行 ID、状态、输入、阶段、最后事件、结果/错误摘要、更新时间。
- 新增 `ContextArchive`：覆盖条目数、结构化摘要、压缩策略、原始/压缩 Token、更新时间。
- 新增 begin/progress/complete/fail/recover 方法；旧 Session JSON 自动兼容。
- `clearSession` 同时清空 checkpoint 和 archive。

### 2. 分层上下文策略

**涉及文件：** `src/core/context.ts`

- microcompact：只在有效视图中缩短较早 Tool Result，原文仍在 Session。
- context collapse：超过条目或 Token 阈值时，将旧完整轮次并入持久 archive。
- full compact：`/compact` 主动归档除最近两轮外的历史。
- PTL truncation：压缩后仍超预算时，继续按完整用户轮次丢弃最早一组。
- 返回 ContextStats，区分 raw/effective/archive token 和实际使用策略。

### 3. 运行生命周期与恢复

**涉及文件：** `src/runtime/mimi-agent.ts`、`src/index.ts`

- 单轮开始自动写 `running` checkpoint。
- 工具/阶段事件更新 checkpoint；成功、失败、Esc 中断分别落盘。
- 创建或切换 Session 时，把遗留 `running` 转为 `interrupted`。
- 恢复提示自动进入动态 instructions。
- `/resume` 合并 checkpoint、Goal、Plan、Team，普通任务无需预先设置 Goal。

### 4. CLI 命令与展示

**涉及文件：** `src/commands.ts`、`src/terminal.ts`、`src/index.ts`

- 新增 `/compact`。
- `/context` 展示原始、有效、归档 Token、策略和压缩时间。
- `/sessions` 标记可恢复会话。
- 切换会话后在 transcript 下显示最近中断点和 `/resume` 提示。

### 5. 文档与验证

**涉及文件：** `README.md`、`docs/ARCHITECTURE.md`、`CHANGELOG.md`、`tests/*.test.ts`

- 覆盖旧 Session 迁移、崩溃恢复、Esc 中断、普通任务 resume、Goal/Team 合并、自动/手动 compact、Tool Pair 不变量和原始 transcript 不变。
- 运行 `npm run check`、`npm test`、`npm run build`。

## 权衡与考量

- 使用规则化结构摘要而非额外模型调用：轻量、确定、无额外成本；未来可在同一 archive 接口增加可选语义总结。
- checkpoint 记录阶段摘要而非完整事件流：Trace 继续承担诊断，checkpoint 只承担恢复。
- 原始 Session 永不因 compact 删除：恢复与审计可靠，代价是磁盘文件会增长，但不会挤占模型上下文。
