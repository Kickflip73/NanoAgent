# Mimi 静默自主巡检实施计划

日期：2026-07-15
状态：已完成
关联调研：`20260715-Mimi静默自主巡检-调研.md`

## 任务目标

让长期自主巡检在没有变化、风险或需要 owner 介入时安静完成，同时保留完整执行证据；直接 command 始终得到回复。

## 方案概述

为每个非 command Dispatcher Run 创建一个轻量内存 `DeliveryControl`，通过单一 Host Tool 标记 suppress。成功提交仍使用现有 Event transaction，仅按控制状态决定是否附带 Outbox。

## UI 变动检测

涉及 UI 变动：否
变动类型：无
预览状态：不适用

## 详细步骤

1. 新增内聚的 delivery tool 模块和有界 reason；command Event 返回空工具集。
2. 注册 `control` tool policy，保持非 side effect、非 SubAgent/Team 能力。
3. Dispatcher 注入工具并在成功时记录 suppression metadata、条件创建 Outbox。
4. 默认 Daily Routines 引导模型只在真的无需关注时静默。
5. 测试 command 隔离、参数边界、策略分类、成功静默、普通自主投递、失败/重试不持久继承。
6. 更新 README、Attention、Architecture、CHANGELOG、总路线图和阶段记录，运行完整 CI。

## 权衡与考量

- 使用显式 Tool，不解析模型文本 sentinel。
- 静默状态只属于当前 attempt，避免新增持久状态和恢复语义。
- 保留 Event/Run 结果，Activity 仍可审计后台工作。
