# 轻量级通用 Agent 升级调研报告

日期：2026-07-13
状态：已审核

## 调研范围

- 目标：将 NanoAgent 从教学示例定位升级为可实际工作的轻量级通用 Agent。
- 涉及模块：运行时、上下文、会话、记忆、计划、Skill、MCP、RAG、工具、CLI、评测与文档。
- 基线：`main` 分支 `078408f`，本地包含尚未提交的 Web Search 与新增 Skills。

## 核心发现

### 现状分析

NanoAgent 已具备可运行的单 Agent 工具循环、流式 CLI、会话、长期记忆、计划、RAG、Skill 和 MCP stdio。项目使用 TypeScript 与 OpenAI Agents SDK，当前 36 项测试、类型检查和 2 项 Retrieval Eval 均通过。

### 关键流程

`CLI -> NanoAgent -> Context/Session/Memory/Plan/Skill/RAG/MCP -> OpenAI Agents SDK Runner -> Tool/MCP -> Stream Events`

### 现有约束

- 保持单进程、本地优先、JSON 持久化，不引入数据库、消息队列或工作流引擎。
- 保持 `nano` 启动命令和现有 CLI 行为兼容。
- OpenAI 与 DeepSeek 均须可用，新增能力不能只依赖 OpenAI Responses 专有状态。
- 当前工作树已有用户改动，必须保留并验证。

### 风险与问题

- `src/agent.ts` 同时承担模型构建、依赖装配、Agent 创建和运行控制，职责偏多。
- Skill 仅用正则解析 YAML，未携带根目录，资源型 Skill 不能可靠解析相对路径。
- MCP 仅支持自定义 `servers` + stdio；单个连接失败会阻断启动。
- 上下文按消息条数裁剪，旧历史摘要只是 JSON 截取，不是真正的结构化压缩。
- Plan 只有步骤列表，缺少 Goal、checkpoint 和跨重启恢复语义。
- 缺少受控 SubAgent、统一生命周期 Hooks 和 Agent 行为评测。

## 与任务相关的关键结论

采用“薄运行时 + 可插拔能力模块”的结构：保留单 Agent 主循环，将模型工厂、提示词、Goal、Skill、MCP、SubAgent 分开；功能通过 SDK 原生 Tool/MCP/Agent-as-tool 接入。Todo 不新增独立系统，而是在现有 Plan 上增加持久 Goal 生命周期。SubAgent 只实现单层 manager-as-tool，避免多 Agent 图和调度框架。
